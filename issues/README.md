# Issues

Lightweight file-based tracking: one directory per milestone, one file per
unit of work, numbered in rough dependency order. Delete a file when its
work ships (git history is the archive).

- `m3-local-testing/` — run challenge tests on the user's machine
- `m4-accounts-hardening/` — account management + web hardening
- `m5-content/` — course content workflow and scoring improvements
- `ops/` — deployment/operations items with no milestone urgency
